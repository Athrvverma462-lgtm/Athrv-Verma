# Food101 CNN — Scratch-Trained Image Classifier

A convolutional neural network trained from scratch on the [Food101](https://data.vision.ee.ethz.ch/cvl/datasets_extra/food-101/) dataset — 101 food categories, 101,000 images.

---

## Project Structure

```
├── food101_cnn.py          # Main training script
├── helper_function.py      # Must supply accuracy_fn
├── data/
│   └── food-101/           # Downloaded by torchvision (see Setup)
└── Food101_CNN/
    └── food101_cnn.pth     # Checkpoint (auto-created on first save)
```

---

## Setup

**1. Install dependencies**

```bash
pip install torch torchvision
```

**2. Download Food101**

Set `download=True` in both `datasets.Food101(...)` calls in `main()` on the first run, then switch back to `False`.

Or download manually:

```bash
# torchvision will place it under data/food-101/
python -c "from torchvision.datasets import Food101; Food101('data', download=True)"
```

**3. Ensure `helper_function.py` is present**

It must expose an `accuracy_fn(y_true, y_pred)` that returns a percentage float. Minimal example:

```python
def accuracy_fn(y_true, y_pred):
    return (y_true == y_pred).float().mean().item() * 100
```

---

## Running

```bash
python food101_cnn.py
```

The script trains one epoch per iteration, prints metrics, then asks:

```
Train for another 1 epoch(s)? (y/n):
```

Type `y` to continue or `n` to stop. The checkpoint is saved after every iteration, so you can safely quit at any point and resume later.

## Architecture — `HighCapacityFood101CNN`

| Layer           | Output shape       | Notes                                                   |
| --------------- | ------------------ | ------------------------------------------------------- |
| Input           | (B, 3, 224, 224)   | RGB, ImageNet-normalised                                |
| Conv Block 1    | (B, 128, 112, 112) | 3→64→128 channels, MaxPool                              |
| Conv Block 2    | (B, 256, 56, 56)   | 128→256→256 channels, MaxPool                           |
| Conv Block 3    | (B, 256, 28, 28)   | 256→256→256 channels, MaxPool                           |
| Conv Block 4    | (B, 512, 14, 14)   | 256→512 channels, MaxPool, Dropout(0.4)                 |
| Global Avg Pool | (B, 512)           | Collapses spatial dims                                  |
| Classifier      | (B, 101)           | Linear(512→256) + ReLU + Dropout(0.3) + Linear(256→101) |

**Total parameters: ~4.7 M**

Each conv layer is followed by ReLU and BatchNorm. BatchNorm stabilises gradient flow and reduces sensitivity to learning rate choice. Global Average Pooling replaces a large fully-connected flatten layer, cutting parameters dramatically while making the model input-resolution agnostic.

## Training Configuration

| Setting       | Value             | Reason                                                  |
| ------------- | ----------------- | ------------------------------------------------------- |
| Optimizer     | AdamW             | Decoupled weight decay vs vanilla Adam                  |
| Learning rate | 3e-4              | Standard AdamW starting point for CNNs                  |
| Weight decay  | 1e-2              | L2 regularisation without distorting adaptive gradients |
| Scheduler     | ReduceLROnPlateau | Halves lr after 3 epochs of no test-loss improvement    |
| Batch size    | 32                | Stable gradient estimates; fits ~4 GB VRAM              |
| Loss function | CrossEntropyLoss  | Standard multi-class classification loss                |

## Data Augmentation

**Train only** (test set uses a clean deterministic crop):

| Transform                      | Purpose                                           |
| ------------------------------ | ------------------------------------------------- |
| `Resize(256)`                  | Standardise input resolution                      |
| `RandomCrop(224)`              | Sees different spatial regions per epoch          |
| `RandomHorizontalFlip()`       | Food is label-invariant under L/R flip            |
| `ColorJitter(0.3, 0.3, 0.2)`   | Handles varying lighting and plating conditions   |
| `Normalize(ImageNet mean/std)` | Centres pixel distribution for faster convergence |

Augmentation is the single highest-impact change for a scratch-trained CNN on Food101 — expect +8–12% accuracy compared to training with only a centre crop.

## Expected Performance

| Epoch | Train acc | Test acc | Test loss |
| ----- | --------- | -------- | --------- |
| 5     | 15–25%    | 12–20%   | 3.5–4.0   |
| 15    | 35–48%    | 30–42%   | 2.5–3.0   |
| 25    | 55–64%    | 47–55%   | 1.7–2.2   |
| 50    | 65–72%    | 52–60%   | 1.4–1.8   |

These are realistic ranges for a **scratch-trained CNN** on Food101. The architecture ceiling is approximately **55–65% top-1** — this is a fundamental limitation of training without pretrained weights on 101 classes, not a bug. A pretrained ResNet50 reaches ~85% on the same dataset.

## Checkpointing

Checkpoints are saved to `Food101_CNN/food101_cnn.pth` after every iteration. They store:

- Model weights (`model_state_dict`)
- Optimizer state (`optimizer_state_dict`) — preserves AdamW momentum buffers
- Current epoch number
- Last training loss

If the checkpoint is incompatible (e.g. after an architecture change), the script detects the mismatch and starts fresh automatically.

> **Note:** delete the checkpoint manually if you change the model architecture, otherwise the load will silently fail and restart from epoch 0.

## Extending This Project

**To push accuracy higher without changing the architecture:**

- Add `transforms.RandomRotation(15)` to train transforms (+2–3%)
- Add `transforms.RandomGrayscale(p=0.1)` for colour invariance

**To break the scratch-CNN ceiling:**

```python
import torchvision.models as models
model = models.resnet50(weights=models.ResNet50_Weights.IMAGENET1K_V1)
model.fc = nn.Linear(2048, 101)   # Replace final layer for Food101
```

Fine-tuning a pretrained ResNet50 reaches ~85% top-1 with the same training loop.
